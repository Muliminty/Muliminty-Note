# 📥 技能：对话知识沉淀 (Capture Knowledge)

## 🎯 技能目标

从当前对话中提取值得长期保留的知识（概念理解、代码实战、问题排错），按知识库规范生成标准笔记文件并写入正确位置。

## 📋 前置规范

执行前必须静默读取以下规范文件，作为输出标准：

- `99-系统/01-百科写作规范.md` — Frontmatter 规范、命名规范、受控后缀词表
- `99-系统/02-百科组织规范.md` — 目录结构、文件职责边界
- `99-系统/模板/01-概念理解类模板.md` — 概念类骨架
- `99-系统/模板/02-实战应用类模板.md` — 实战类骨架
- `99-系统/模板/03-问题排错类模板.md` — 排错类骨架

## 🧠 知识提取规则

### 1. 识别可沉淀知识

从对话中提取符合以下条件的内容：

- 对某个概念建立了清晰理解（定义、原理、架构）
- 产出了可复用的代码模式、配置方案、命令组合
- 解决了一个具体技术问题（报错、排查、修复）
- 形成了可复用的决策依据或最佳实践

**不应沉淀的内容**：

- 纯闲聊、项目管理讨论
- 已有笔记覆盖的重复知识
- 过于碎片化、缺乏独立价值的信息

### 2. 判断知识类型

| 类型 | 判断信号 | 对应模板 |
|------|----------|----------|
| 概念理解 | "是什么"、"为什么"、原理、架构、对比 | `01-概念理解类模板.md` |
| 实战应用 | "怎么做"、代码实现、配置方案、环境搭建 | `02-实战应用类模板.md` |
| 问题排错 | 报错日志、排查过程、Bug 修复 | `03-问题排错类模板.md` |

不确定时，默认使用概念理解类模板。

### 3. 判断归属目录

```
判断流程：
1. 知识是否明确属于某个已有专题？
   → 是：写入该专题目录，使用完整 frontmatter（10 字段）
   → 否：进入第 2 步
2. 知识是否是独立原子概念？
   → 是：写入 98-知识库/，使用精简 frontmatter（3 字段）
   → 否：写入 98-知识库/，后续人工归类
```

## 📝 输出规范

### 写入 `98-知识库/` 时

文件名：`YYYYMMDD-领域-核心概念.md`

Frontmatter（精简模式，仅 3 个必填字段）：

```yaml
---
title: "核心概念名"
date: "YYYY-MM-DD"
tags: ["领域标签"]
---
```

### 写入正式专题目录时

文件名：`[两位序号]-[主题名].md`

Frontmatter（完整模式）：

```yaml
---
title: "文章标题"
date: "YYYY-MM-DD"
lastModified: "YYYY-MM-DD"
status: "草稿"
tags: ["标签1", "标签2"]
moc: "[[!MOC-分类名称]]"
stage: "基础入门"
prerequisites: [""]
description: "文章描述"
aliases: []
toc: true
---
```

### 正文要求

- 使用对应模板的骨架结构
- 内容源自对话，但需要重新组织为结构化笔记，不是对话记录的复制粘贴
- 遵循"通俗易懂、由浅入深"的写作口径
- 中英文混排保留空格，技术名使用官方写法

## 🚀 执行流程

当用户说"沉淀"、"记下来"、"写入笔记"或类似指令时：

### 步骤 1：提取与确认

1. 回顾当前对话，提取可沉淀的知识点（通常 1-3 个）
2. 向用户展示提取结果，包括：
   - 知识点标题
   - 知识类型（概念/实战/排错）
   - 目标路径（`98-知识库/` 或具体专题目录）
3. 等待用户确认或调整

**示例输出**：

```
本次对话可沉淀以下知识：

1. 📥 Docker 多阶段构建原理
   - 类型：概念理解
   - 路径：98-知识库/20260509-容器-Docker多阶段构建原理.md

2. 📥 Nginx 反向代理跨域配置
   - 类型：实战应用
   - 路径：07-基础设施与运维/02-Web服务器/Nginx/05-反向代理跨域配置.md

是否确认？可以调整类型、路径或去掉不需要的条目。
```

### 步骤 2：读取规范与模板

用户确认后，静默读取：
- 对应的模板文件
- 目标目录下的 `!MOC` 和 `README.md`（如果写入正式专题）

### 步骤 3：生成并写入

1. 按模板骨架生成笔记内容
2. 写入目标路径
3. 如果写入正式专题，检查对应 `!MOC` 是否需要补充索引链接

### 步骤 4：报告

输出写入结果：
- 文件路径
- Frontmatter 摘要
- 如有联动更新（MOC 等），一并说明

## ⚠️ 约束规则

- **一次对话建议沉淀 ≤ 3 篇**；超过时提醒用户后续批量整理，用户确认后可突破上限
- **不确定归属时，必须默认写入 `98-知识库/`**，不得自行创建新的专题目录
- **用户拒绝沉淀时，不再追问**，本次对话内不重复提议同一知识点
- **严禁修改** `quartz.config.ts` 和 `index.md`
- **标签使用中文领域标签**，遵守 `01-百科写作规范.md` §3.2 标签受控规则
- **文件名遵守受控后缀词表**，避免使用"总结"、"整理"、"踩坑"等黑名单词
