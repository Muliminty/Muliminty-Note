# 部署与发布 MOC

> 前端应用的部署和发布流程，包括静态站点托管、CI/CD、发布策略。
> 
> **学习路径**：部署与发布是项目开发的最后阶段，需要掌握 [构建工具](../../03-工程化实践/工具链与构建/!MOC-工具链与构建.md) 和 [工程化实践](../../03-工程化实践/工程化/!MOC-工程化.md)。部署策略与 [架构设计](../架构/!MOC-架构.md) 和 [性能优化](../../04-质量保障/性能/!MOC-性能.md) 相关。

---

## 📚 核心主题

- [静态站点托管（Netlify/Vercel）](./静态站点托管.md) — 静态站点部署（配合 [构建工具](../../03-工程化实践/工具链与构建/!MOC-工具链与构建.md) 使用）
- [CI/CD流程（GitHub Actions/GitLab CI）](./CI-CD流程.md) — CI/CD 自动化（配合 [测试](../../04-质量保障/测试/!MOC-测试.md) 使用）

## 📝 学习建议

1. **前置知识**：需要掌握 [构建工具](../../03-工程化实践/工具链与构建/!MOC-工具链与构建.md) 和 [工程化实践](../../03-工程化实践/工程化/!MOC-工程化.md)
2. **学习顺序**：构建工具 → 工程化 → 部署基础 → CI/CD → 发布策略
3. **相关主题**：
   - [构建工具](../../03-工程化实践/工具链与构建/!MOC-工具链与构建.md)：构建产物用于部署
   - [测试](../../04-质量保障/测试/!MOC-测试.md)：CI/CD 中集成测试
   - [性能优化](../../04-质量保障/性能/!MOC-性能.md)：部署时考虑性能优化
   - [架构设计](../架构/!MOC-架构.md)：部署架构设计
4. **实践应用**：部署策略需要根据项目规模和需求选择，大型项目需要考虑灰度发布、回滚等策略
