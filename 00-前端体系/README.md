## 🎯 设计原则

### 1. 分层清晰
- **01-核心语言**：基础语言（HTML、CSS、JavaScript）
- **02-框架与库**：基于语言之上的框架和库
- **03-工程化**：开发工具和流程
- **04-性能与优化**：性能相关
- **05-安全与合规**：安全和法规
- **06-网络与数据**：网络和数据交互
- **07-架构与设计**：架构和设计模式
- **08-部署与运维**：部署和运维
- **09-运营与协作**：运营和团队协作
- **10-前沿与资源**：前沿技术和学习资源

### 2. 按技术栈分类
- **框架与库** 目录下按框架分类（React、Vue、Angular）
- **状态管理** 目录下按框架分类（React 生态、Vue 生态）
- **工具链** 目录下按工具分类（Vite、Webpack 等）

### 3. 易于扩展
- 使用数字前缀便于排序和扩展
- 每个目录都可以独立扩展子目录
- 保持一致的命名规范

## 📝 命名规范

### 目录命名
- 使用 `数字-分类` 格式：`01-核心语言`、`02-框架与库`
- 数字前缀便于排序和扩展
- 分类名使用中文，便于理解

### 文件命名
- MOC 文件：`!MOC-分类.md`
- 普通文件：使用中文或英文，中划线分隔
- 示例：`Redux-Saga.md`、`状态管理基础.md`

### 链接规范
- 使用相对路径：`./状态管理/Redux.md`
- 保持链接的可读性

## 📌 扩展指南

### 添加新内容

1. **确定分类**：属于哪个大类？
2. **选择目录**：找到对应的数字前缀目录
3. **创建文件**：使用规范的文件名
4. **更新索引**：在对应的 MOC 文件中添加链接

### 添加新框架

1. **在 `02-框架与库/` 下创建新目录**：如 `Svelte/`
2. **创建 MOC 文件**：`!MOC-Svelte.md`
3. **在主 MOC 中添加链接**

### 添加新工具

1. **确定分类**：构建工具 → `03-工程化/工具链与构建/`
2. **创建文件**：使用规范的文件名
3. **更新索引**：在对应的 MOC 文件中添加链接
