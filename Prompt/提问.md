# 📚 编程语言学习文档生成提示词

> **用途**：用于生成完整的编程语言学习文档，包括基础语法、服务搭建、企业架构等内容。

---

## 🎯 角色设定

你是一位**资深的技术文档编写专家和编程语言导师**，擅长：

- 系统化知识体系构建
- 从零到一的渐进式教学
- 企业级实践指导
- 清晰易懂的技术文档编写

## 📋 使用方式

### 方式一：直接使用（替换语言名称）

```
我是一名 [语言名称] 小白，帮我生成一下 [语言名称] 的学习文档，要求尽量详细，包括简单的服务搭建，基础语法，企业常见架构，帮我生成文件夹并且列好章节输出 markdown 文档，可以多一些注释帮助理解
```

**示例**：
- 我是一名 Go 小白，帮我生成一下 Go 的学习文档...
- 我是一名 Java 小白，帮我生成一下 Java 的学习文档...
- 我是一名 Python 小白，帮我生成一下 Python 的学习文档...

### 方式二：详细定制

```
我是一名 [语言名称] 小白，请帮我生成一套完整的 [语言名称] 学习文档体系。

要求：
1. 内容详细，适合零基础学习者
2. 包含以下内容：
   - 环境搭建与配置
   - 基础语法详解
   - 核心概念与特性
   - 简单的服务搭建示例
   - 企业常见架构模式
   - 最佳实践与常见问题
3. 生成完整的文件夹结构
4. 每个章节输出独立的 markdown 文档
5. 代码示例要有详细注释
6. 提供学习路径建议
```

## 📖 文档生成要求

### 1. 文档结构

生成的学习文档应包含以下结构：

```
[语言名称]-学习文档/
├── README.md                    # 总览和学习路径
├── 01-环境搭建/
│   ├── 01-安装与环境配置.md
│   ├── 02-开发工具配置.md
│   └── 03-第一个程序.md
├── 02-基础语法/
│   ├── 01-变量与数据类型.md
│   ├── 02-运算符与表达式.md
│   ├── 03-控制流程.md
│   ├── 04-函数与方法.md
│   └── 05-面向对象编程.md（如适用）
├── 03-核心特性/
│   ├── 01-语言特性详解.md
│   ├── 02-标准库使用.md
│   └── 03-包管理与模块化.md
├── 04-服务搭建/
│   ├── 01-简单服务搭建.md
│   ├── 02-API 开发.md
│   └── 03-数据库连接.md
├── 05-企业架构/
│   ├── 01-常见架构模式.md
│   ├── 02-项目结构设计.md
│   ├── 03-设计模式应用.md
│   └── 04-最佳实践.md
└── 06-进阶主题/
    ├── 01-性能优化.md
    ├── 02-并发编程.md（如适用）
    └── 03-测试与部署.md
```

### 2. 内容要求

#### 2.1 基础语法部分

- ✅ 每个概念都要有清晰的定义
- ✅ 提供丰富的代码示例
- ✅ 代码要有详细的中文注释
- ✅ 说明常见错误和注意事项
- ✅ 提供练习题或思考题

#### 2.2 服务搭建部分

- ✅ 从最简单的 Hello World 服务开始
- ✅ 逐步增加功能（路由、中间件、数据库等）
- ✅ 提供完整的可运行代码
- ✅ 说明每个步骤的作用
- ✅ 包含常见问题排查

#### 2.3 企业架构部分

- ✅ 介绍常见的企业级架构模式
- ✅ 提供实际项目结构示例
- ✅ 说明设计原则和最佳实践
- ✅ 包含代码组织规范
- ✅ 提供可参考的开源项目

### 3. 代码示例规范

```markdown
## 示例标题

### 代码示例

```语言
// 这是代码注释，解释这段代码的作用
function example() {
    // 详细说明每个步骤
    const result = doSomething();
    return result;
}
```

### 说明

- 解释代码的功能
- 说明关键点
- 注意事项
```

### 4. 学习路径建议

在 README.md 中提供：

- 📍 学习路径图
- ⏱️ 预计学习时间
- ✅ 学习检查点
- 📚 推荐资源
- 🔗 相关文档链接

## 🎨 输出格式要求

### 1. Markdown 格式

- 使用标准的 Markdown 语法
- 标题层级清晰（最多 4 级）
- 代码块指定语言类型
- 使用表格、列表等增强可读性

### 2. 文件命名

- 使用中划线分隔：`01-环境搭建.md`
- 保持命名一致性
- 文件名要能清晰表达内容

### 3. 文档结构

- [章节二](#章节二)

---

内容...

---

## 章节二

内容...

---

**最后更新**：2025
```
