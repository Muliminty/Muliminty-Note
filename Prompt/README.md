# 📝 Prompt 提示词仓库

> 本目录包含用于辅助 Cursor 等 AI 工具的提示词模板，帮助提高开发效率和文档质量。

---

## 📋 目录

- [仓库规范文档.md](./仓库规范文档.md) - 知识库组织规范和维护指南
- [脚手架.md](./脚手架.md) - 前端脚手架构建指导提示词
- [提问.md](./提问.md) - 编程语言学习文档生成提示词
- [英文技术文档翻译.md](./英文技术文档翻译.md) - 英文技术文档翻译提示词

## 🎯 使用说明

### 1. 仓库规范文档

**用途**：了解知识库的组织规范、文件命名、内容格式等。

**适用场景**：
- 新增笔记时参考规范
- 维护知识库结构
- 统一文档格式

**使用方式**：
- 直接阅读参考
- 在 Cursor 中引用相关章节

---

### 2. 脚手架构建提示词

**用途**：指导 AI 从零构建一个可扩展的前端脚手架系统。

**适用场景**：
- 需要创建类似 Vite / create-react-app 的脚手架
- 学习前端工程化知识
- 构建自定义项目模板

**使用方式**：
1. 复制 `脚手架.md` 中的提示词
2. 在 Cursor 中粘贴使用
3. 根据需求调整功能要求

**示例**：
```
请按照脚手架构建提示词，帮我创建一个支持 React + TypeScript 的脚手架
```

---

### 3. 学习文档生成提示词

**用途**：生成完整的编程语言学习文档体系。

**适用场景**：
- 需要为某门编程语言创建学习文档
- 系统化整理学习笔记
- 构建知识体系

**使用方式**：
1. 复制 `提问.md` 中的提示词模板
2. 替换语言名称（如 Go、Java、Python 等）
3. 在 Cursor 中使用

**示例**：
```
我是一名 Go 小白，帮我生成一下 Go 的学习文档，要求尽量详细，包括简单的服务搭建，基础语法，企业常见架构，帮我生成文件夹并且列好章节输出 markdown 文档，可以多一些注释帮助理解
```

---

### 4. 英文技术文档翻译提示词

**用途**：将英文技术文档翻译为高质量的中文技术文档。

**适用场景**：
- 翻译官方文档
- 翻译技术博客
- 翻译 API 文档

**使用方式**：
1. 复制 `英文技术文档翻译.md` 中的提示词
2. 在提示词末尾粘贴要翻译的英文文档
3. 在 Cursor 中执行翻译

**示例**：
```
请将以下英文技术文档翻译为中文：

[在此粘贴英文文档]
```

## 💡 最佳实践

### 1. 提示词定制

- 根据具体需求调整提示词内容
