# ✅ **前端脚手架构建完整 AI 提示词（可直接使用）**

**用途：让 AI 从 0 到 1 指导我构建一个可扩展的前端脚手架，并实现我自定义模板功能。**

---

**提示词开始：**

请你充当一个拥有资深架构经验的前端工程化专家，你的任务是**从零帮助我构建一个类似 Vite / create-react-app 的前端脚手架系统**。你的讲解和指导必须满足以下要求：

---

## **（1）输出结构要求**

每次回答必须包含以下内容：

1. **知识讲解**：讲清楚每一部分背后的原理，包括

   * 前端脚手架是什么
   * 运行机制
   * Node CLI 工作原理
   * 文件模板处理
   * 插件架构
   * 扩展机制

2. **从 0 开始步骤拆解**
   从“创建一个空目录”开始，逐步搭建出完整脚手架，包括：

   * 使用 Node / TS 作为执行环境
   * 通过 commander / yargs 处理命令
   * 实现模版创建（ejs / mustache）
   * 实现用户交互选择（inquirer）
   * 内置模版体系（Vue / React / TS / 工具库 模板）
   * 插件机制（可选）
   * 兼容 npm / pnpm / yarn
   * 文件生成与动态注入
   * 脚手架发布到 npm

3. **代码示例**
   给出完整可直接粘贴运行的代码，包含：

   * package.json
   * bin 入口文件
   * cli.ts
   * template 管理器
   * config 文件
   * 插件扩展示例
   * 模板 demo（例如 React + TS）

4. **实际运行效果展示**
   包括我执行以下时会看到什么：

   ```bash
   my-cli create
   my-cli create my-app
   my-cli create --template vue-ts
   ```

5. **进一步扩展能力**
   引导我继续扩展脚手架，例如：

   * 添加插件系统
   * 增加 config 文件
   * 允许用户创建自己的模板
   * 支持下载远程模板（GitHub）
   * 支持模板变量注入
   * 添加 mock 服务器
   * 添加运行时能力（类似 Vite 中间件）

---

## **（2）脚手架目标功能（请按以下要求构建）**

我要最终拥有一个具备以下能力的脚手架：

### **核心功能**

* `my-cli create <project-name>`
  自动生成项目目录

* 模板系统（模板类型可扩展）

  * basic-js
  * basic-ts
  * vue
  * vue-ts
  * react
  * react-ts

* 支持用户交互：

  * 是否安装依赖
  * 选择包管理器
  * 是否初始化 git
  * 是否添加 ESLint / Prettier / Husky

* 支持远程模板：

  * 例如 GitHub 仓库
  * zip 包
  * npm 包

* 支持插件机制

  * my-cli plugin add xxx
  * 类似 Vite 插件体系的 Hook 示例

### **工程化能力**

* 可选择使用 TypeScript
* 完整的目录结构设计
* 代码分层（commands / utils / templates / core）
* 错误处理
* 日志系统
* 自动测试（Jest）
* 发布到 npm 的流程（含步骤）

---

## **（3）输出风格要求**

* 所有内容使用 **清晰结构化** 的方式输出
* 保证每一步讲解 + 示例 + 注意事项
* 不要跳步
* 默认使用 TypeScript
* 所有示例必须可运行
* 必须保持逻辑连贯性，从易到难
