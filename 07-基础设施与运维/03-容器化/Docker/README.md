# Docker 目录说明

本目录用于承载 Docker 相关的正式知识内容，覆盖从基础入门、镜像与容器管理，到 Docker Compose、生产部署与故障排查的完整主线。

## 1. 收录范围

本目录优先收录：

- Docker 基础概念、安装与常用命令
- 镜像构建、Dockerfile 编写与镜像优化
- 容器运行、网络、存储与资源管理
- Docker Compose 多服务编排
- 生产部署、监控、日志与故障排查

本目录不优先收录：

- 临时实验记录
- 未整理完成的碎片化指令摘录
- 仅作历史保留的旧笔记

上述非主干内容统一进入 [99-参考与归档](./99-参考与归档/)。

## 2. 当前结构

当前专题树按知识角色划分为以下九个正式分区：

- `01-基础入门`
- `02-镜像管理`
- `03-容器管理`
- `04-Dockerfile高级应用`
- `05-Docker-Compose`
- `06-网络与存储`
- `07-安全与最佳实践`
- `08-生产环境部署`
- `09-故障排查`

入口文件职责：

- [!MOC-Docker.md](./!MOC-Docker.md) 承担主题导航与学习路径
- `README.md` 只解释目录结构与维护方式

## 3. 阅读建议

建议按以下顺序学习：

1. [01-基础入门/01-安装与环境配置.md](./01-基础入门/01-安装与环境配置.md)
2. [01-基础入门/02-核心概念.md](./01-基础入门/02-核心概念.md)
3. [01-基础入门/03-基本命令.md](./01-基础入门/03-基本命令.md)
4. [02-镜像管理](./02-镜像管理/)
5. [03-容器管理](./03-容器管理/)
6. [05-Docker-Compose](./05-Docker-Compose/)
7. [08-生产环境部署](./08-生产环境部署/)
8. [09-故障排查](./09-故障排查/)

如果是按场景查阅：

- 开发构建：看 `02-镜像管理`、`04-Dockerfile高级应用`
- 日常运行：看 `03-容器管理`、`06-网络与存储`
- 多服务项目：看 `05-Docker-Compose`
- 上线与运维：看 `08-生产环境部署`、`09-故障排查`

## 4. 维护约束

- 新增正文统一使用编号文件命名
- 目录级说明写入 `README.md`，不混入主题导航
- 学习路径与主题索引统一维护在 [!MOC-Docker.md](./!MOC-Docker.md)
- 旧版材料、指令摘录与历史方案统一收敛到 [99-参考与归档](./99-参考与归档/)
- 若后续新增子专题，应优先判断是否属于现有九个分区，避免继续在根目录平铺
