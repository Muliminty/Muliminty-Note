# Nginx 性能优化目录说明

本目录用于收纳 Nginx 在吞吐、连接、缓存、压缩与资源利用方面的优化主题。

## 1. 本目录负责什么

本目录重点回答三个问题：

- Nginx 当前容量瓶颈更可能出现在哪一层
- 常见性能参数应该如何理解与调优
- 如何用更低成本的方式提升吞吐、稳定性和资源利用率

优先收录：

- worker 进程与连接数调优
- 缓存、压缩与静态资源加速
- 超时、缓冲区与内核参数调优
- 压测与性能观测方法

不优先收录：

- 基础静态文件服务
- 单纯的配置语法说明
- 故障排查记录

## 2. 当前结构

1. [01-worker与连接调优.md](./01-worker与连接调优.md)
2. [02-压缩缓存与静态资源加速.md](./02-压缩缓存与静态资源加速.md)
3. [03-超时与缓冲区调优.md](./03-超时与缓冲区调优.md)

## 3. 推荐阅读顺序

建议按下面顺序阅读：

1. 先看 worker 与连接调优，建立容量认知。
2. 再看压缩缓存与静态资源加速，掌握最常见也最容易见效的优化手段。
3. 最后看超时与缓冲区调优，理解反向代理链路中的性能取舍。

## 4. 维护约束

- 这里优先收录可复用的性能方法，不承载单次事故复盘。
- 如果内容核心是“系统为什么报错、怎么定位”，优先考虑 `09-故障排查` 或 `07-日志与监控`。
- 如果内容核心是“缓存与静态资源服务的基础能力”，优先与 `03-核心功能` 协同，而不是重复铺写。
