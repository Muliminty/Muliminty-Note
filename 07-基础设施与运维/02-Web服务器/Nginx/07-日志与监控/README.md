# Nginx 日志与监控目录说明

本目录用于收纳 Nginx 的访问日志、错误日志、状态监控、指标采集与基础可观测性主题。

## 1. 本目录负责什么

本目录重点回答三个问题：

- 请求记录是否足够支撑排障与分析
- Nginx 运行时是否出现了错误与异常趋势
- 哪些基础指标可以帮助我们尽早发现问题

优先收录：

- 日志格式与日志切分
- 错误日志定位方法
- 状态监控与指标采集
- 告警、健康检查与排障观测

不优先收录：

- 具体业务故障复盘
- 纯部署案例
- 单一配置项的基础解释

## 3. 推荐阅读顺序

建议按下面顺序阅读：

1. 先看访问日志，建立“请求如何被记录”的基础认知。
2. 再看错误日志，掌握最常见的定位入口。
3. 最后看状态监控与指标采集，把日志视角补成实时观测视角。

## 4. 维护约束

- 这里优先收录“通用方法”，不承载一次性的事故复盘。
- 如果内容核心是“某个线上问题怎么定位和修复”，优先考虑 `09-故障排查`。
- 如果内容核心是“安全限制、风控、防护”，优先考虑 `06-安全`。
