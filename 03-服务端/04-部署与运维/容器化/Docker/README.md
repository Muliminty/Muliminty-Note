## 🎯 设计原则

### 1. 循序渐进

- **阶段一**：基础入门，掌握 Docker 基本概念和命令
- **阶段二**：镜像与容器管理，能够构建和管理镜像容器
- **阶段三**：多服务编排，使用 Docker Compose 管理多服务应用
- **阶段四**：生产环境应用，在生产环境中部署和管理 Docker 应用

### 2. 理论与实践结合

- 每个知识点都包含概念说明和实践示例
- 提供实际项目案例和最佳实践
- 包含常见问题和故障排查方法

### 3. 完整知识体系

- 覆盖 Docker 的完整生命周期
- 从开发到部署到运维的全流程
- 包含安全、优化、监控等各个方面

### 如何学习

1. **从 MOC 开始**：查看 [Docker 知识体系（MOC）](./!MOC-Docker.md) 了解完整知识结构
2. **按阶段学习**：遵循 MOC 中的学习路径，循序渐进
3. **实践为主**：每个知识点都要动手实践，加深理解
4. **查阅文档**：遇到问题时查阅对应的详细文档

### 如何添加新内容

1. 在对应的分类目录下创建新的 Markdown 文件
2. 在 `!MOC-Docker.md` 中添加链接
3. 在文件末尾添加双链回到 MOC：`[[!MOC-Docker|返回 Docker 知识体系]]`
4. 添加适当的标签

### 文件命名规范

- **MOC 文件**：`!MOC-分类名.md`（如 `!MOC-Docker.md`）
- **普通文件**：使用中划线分隔（如 `安装与环境配置.md`）
- **目录名**：使用数字前缀+中划线+名称（如 `01-基础入门`）
