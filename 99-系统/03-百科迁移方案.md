# 百科迁移方案

## 1. 目标

本次迁移的目标，是将当前仓库从历史演化出的多入口结构，逐步迁移为统一的“个人技术百科”。

迁移期间允许新旧目录并存，但所有新增内容应尽量直接写入目标结构。

目标一级目录如下：

- `00-总览与索引`
- `01-计算机基础`
- `02-编程语言`
- `03-前端开发`
- `04-服务端开发`
- `05-数据与存储`
- `06-架构与工程`
- `07-基础设施与运维`
- `08-AI与智能开发`
- `09-工具与效率`
- `10-实战案例`
- `11-问题排错`
- `98-知识库`
- `99-系统`

---

## 2. 一级目录迁移映射

主要拆分至：

- `02-编程语言`
- `03-前端开发`
- `06-架构与工程`

建议映射：

- `JavaScript` -> `02-编程语言/01-JavaScript`
- `TypeScript` -> `02-编程语言/02-TypeScript`
- `HTML`、`CSS` -> `03-前端开发/01-Web基础`
- `02-框架进阶` -> `03-前端开发/02-前端框架`
- `03-工程化实践` -> `03-前端开发/03-工程化`
- `04-质量保障` -> `03-前端开发/04-质量保障`
- `05-高级应用` -> `03-前端开发/05-高级应用`
- `07-团队协作` -> `06-架构与工程/07-团队协作`

### 2.2 `00-经验技巧`

主要拆分至：

- `09-工具与效率`
- `10-实战案例`
- `11-问题排错`
- `07-基础设施与运维`

建议映射：

- `01-前端实践` -> `10-实战案例/01-前端实战`
- `02-开发工具` -> `09-工具与效率/01-开发工具`

### 2.8 `06-BLOG`

主要拆分至：

- `10-实战案例`
- `09-工具与效率`
- `08-AI与智能开发`
- `00-总览与索引`

原则：

- 知识型文章拆回正式主题
- 案例型文章进入 `10-实战案例`
- 工具型文章进入 `09-工具与效率`
- 纯时间流内容视情况进入 `99-参考与归档`

## 3. 分阶段推进顺序

建议按以下顺序推进：

1. 建立新一级目录骨架与总入口文件
2. 迁移边界清晰专题：`数据库`、`Docker`、`Nginx`
3. 迁移语言骨干专题：`JavaScript`、`TypeScript`
4. 拆分 `00-经验技巧`
5. 合并 `04-AI与机器学习` 与 `05-AI辅助开发`
6. 拆分或吸收 `06-BLOG`

---

## 4. 执行规则

迁移期间统一遵循：

1. 每次只处理一个一级主题或一个成熟子专题。
2. 每次先列旧路径到新路径映射，再执行移动。
3. 每次迁移后同步修正 `!MOC`、`README.md` 与正文链接。
4. 若文档同时适合多个一级目录，优先按主语义归位，其余通过链接解决。
5. 所有进度以 [04-重构TODO.md](./04-重构TODO.md) 为唯一执行清单。

补充收敛要求：

- `归档`、`!小记`、`收集箱` 一律优先收敛为 `99-参考与归档`
- 历史导航页一律迁入 `99-参考与归档`
- 以上收敛目录默认不参与站点发布
