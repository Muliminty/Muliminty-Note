# 百科写作规范

## 1. 目标

本规范用于统一知识库正文文件的写法，确保所有文档具备以下特征：

- 可被稳定渲染
- 可被稳定检索
- 可被稳定迁移
- 可被脚本稳定处理

本规范适用于正式体系文章、原子知识、系统规范文档与案例文档。

---

## 2. Markdown 与链接规范

### 2.1 原生 Markdown 优先

统一使用标准 Markdown 语法，不依赖特定编辑器的私有写法。

链接统一使用：

```md
[展示文本](相对路径)
```

目标：

- 保证 Obsidian、Quartz 与其他 Markdown 工具兼容
- 保证后续迁移与静态站点构建稳定

### 2.2 双链表达口径

知识关系优先通过正常链接和 `moc` 字段表达，不用随意生成无归属的孤立链接。

要求：

- 文章应尽量挂靠某个 `!MOC`
- 正式文章应尽量补充前置知识
- 跨专题引用要明确主语义，不要把链接堆成标签云

---

## 3. Frontmatter 规范

所有正式文章顶部必须包含 YAML Frontmatter。

推荐模板如下：

```yaml
---
title: "文章标题"
date: "YYYY-MM-DD"
lastModified: "YYYY-MM-DD"
tags: ["标签1", "标签2"]
moc: "[[!MOC-分类名称]]"
description: "文章描述"
publish: true
aliases: ["文章别名1", "文章别名2"]
toc: true
---
```

说明：

- `title`：作为文档唯一识别名
- `date`：首次创建时间
- `lastModified`：最后维护时间
- `tags`：知识分类标签
- `moc`：所在主题的入口锚点
- `description`：摘要说明
- `publish`：是否允许发布
- `aliases`：异名检索补充
- `toc`：是否显示大纲

补充建议：

- 若文章依赖明显前置知识，可增加 `prerequisites`
- 若文章处于草稿期，可将 `publish` 设为 `false`

## 4. 文件命名规范

### 4.1 总原则

文件命名遵循三个原则：

- `受控`
- `稳定`
- `可插队`

也就是：

- 不自由发挥命名风格
- 同层长期保持一致
- 后续可以平滑插入新文件

### 4.2 正式体系文章

正式专题文章统一使用：

```text
[两位序号]-[主题名].md
```

示例：

- `01-安装与环境配置.md`
- `02-核心概念.md`
- `03-多服务编排实践.md`
- `04-Compose文件编写指南.md`

约束：

- 文件名只表达当前层主题
- 不重复上级目录已经表达的信息
- 同层统一使用两位数字前缀

### 4.3 原子知识

`98-知识库` 中的原子文件统一使用：

```text
[YYYYMMDD]-[领域]-[核心概念].md
```

示例：

- `20260315-网络-HTTP缓存协商机制.md`
- `20260315-架构-幂等设计模式.md`
- `20260315-效率-代码审查检查清单.md`

### 4.4 特殊文件

- `!MOC-[主题名].md`：主题入口与索引
- `README.md`：目录说明

## 5. 技术名与混排规范

技术名统一采用官方写法：

- `JavaScript`
- `TypeScript`
- `Node.js`
- `Next.js`
- `Nginx`
- `Docker`
- `Claude Code`
- `VSCode`
- `GitHub`

中英文混排时，英文技术词与中文说明之间保留空格。

推荐：

- `React Hooks 原理`
- `Docker Compose 基础`
- `Windows 系统重装与环境初始化清单`

不推荐：

- `ReactHooks原理`
- `Docker-Compose基础`
- `Windows系统重装与环境初始化清单`

---

## 6. 受控后缀词表

为减少同义异名，优先从以下后缀中选词：

概念类：

- `基础`
- `原理`
- `机制`
- `架构`

使用类：

- `快速开始`
- `配置指南`
- `使用教程`

实战类：

- `实战`
- `实践`
- `最佳实践`

排错类：

- `问题排查`
- `故障诊断`
- `修复方案`

对比类：

- `对比分析`

速查类：

- `速查`
- `清单`

尽量避免：

- `总结`
- `整理`
- `踩坑`
- `终极版`
- `大全`
- `手册`

---

## 7. 禁止项

以下命名与写法应尽量避免：

- 错别字命名，如 `ngnix`
- 大小写漂移，如 `nginx` 与 `Nginx` 混用
- 情绪化命名，如“保姆级”“终极版”
- 目录和文件重复表达同一信息
- 同层同时混用有序与无序命名

---

## 8. 模板使用原则

正文内容按知识类型选择模板：

- 概念理解类
- 实战应用类
- 问题排错类

模板入口见：

- [模板/00-模板使用指南.md](./模板/00-模板使用指南.md)
- [模板/01-概念理解类模板.md](./模板/01-概念理解类模板.md)
- [模板/02-实战应用类模板.md](./模板/02-实战应用类模板.md)
- [模板/03-问题排错类模板.md](./模板/03-问题排错类模板.md)

补充约束：

- 模板仅用于正式正文文件。
- `!MOC` 与 `README.md` 不套用正文模板。
- `99-参考与归档` 中的历史资料不强制补齐正文模板。
