# 重构 TODO

## 1. 使用说明

本文件用于追踪知识库重构进度，后续每完成一项整理动作，都在对应事项后打勾。

执行原则：

- 保留现有大目录骨架，不进行推倒重来式重构。
- 优先复用老分层，在原有层级上逐步规范化。
- 先整理目录结构与文件命名，再处理正文内容。
- 每次只处理一个专题，避免全库同时大改。
- 碎片内容优先收敛，避免继续新增临时目录。

## 5. 单轮重构执行模板

以后每处理一个专题，都按以下顺序勾选：

- [ ] 盘点当前目录树
- [ ] 标记非标准目录
- [ ] 统一目录命名
- [ ] 统一文件命名
- [ ] 补齐 `!MOC`
- [ ] 补齐 `README`
- [ ] 收敛碎片内容
- [ ] 复查是否仍有旧路径残留
- [ ] 记录本轮已完成事项

## 6. 每轮结束后的复盘清单

- [ ] 新目录是否比旧目录更稳定
- [ ] 命名是否比之前更统一
- [ ] 是否减少了临时目录
- [ ] 是否减少了模糊文件名
- [ ] 是否保留了原有知识体系可读性
- [ ] 是否为下一轮整理留下清晰边界
