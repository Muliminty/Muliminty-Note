#  语法规范
1. **语法与链接 (原生强兼容)**
   严格使用纯正的 Markdown 语法
2. **目录与层级 (体系化存储)**
   生成文档严格对齐现有目录树；碎片化概念需收敛至专属“98-知识库”并打满标签。
3. **内容创作骨架 (逻辑闭环)**
   拒绝空洞陈述。强制基于现有的“官方内容模板”（概念类/实战类/排错类）进行垂直输出。
4. **资产稳健命名 (高纯净度保障)**
   文件名一律剔除特殊字符与空格，统一使用中划线 `-`，遵循系统可扩展排序规则。
5. **系统可持续演进 (机器可读)**
   格式排版必须严谨、易于正则提取，为未来的自动化持续维护打好基础。
