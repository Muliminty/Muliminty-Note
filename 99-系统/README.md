# 系统目录说明

`99-系统` 是整个知识库的治理层目录，用于集中维护写作规范、组织规则、迁移策略、执行清单与模板资产。

本目录的核心目标不是承载知识正文，而是约束“知识应如何被书写、归档、迁移与维护”。

## 1. 目录职责

本目录负责：

- 统一知识库的 Markdown、Frontmatter、命名与链接口径
- 统一目录层级、入口文件与归档区的角色边界
- 定义全库重构迁移的目标结构与执行顺序
- 提供概念理解类、实战应用类、问题排错类模板

本目录不负责：

- 承载具体技术主题正文
- 替代专题 `!MOC` 或专题 `README.md`
- 长期堆放临时笔记、剪藏资料与历史草稿

## 2. 结构说明

当前 `99-系统` 目录由三类资产组成：

### 2.1 主规范文件

- [01-百科写作规范.md](./01-百科写作规范.md)
  约束 Markdown 语法、Frontmatter、命名规则、技术名写法与受控表达。
- [02-百科组织规范.md](./02-百科组织规范.md)
  约束一级目录目标结构、专题树骨架、`!MOC`、`README.md`、`98-知识库` 与 `99-参考与归档` 的职责边界。
- [03-百科迁移方案.md](./03-百科迁移方案.md)
  定义旧目录到新百科结构的迁移方向、阶段顺序与执行规则。
- [04-重构TODO.md](./04-重构TODO.md)
  作为全库重构的唯一执行清单，持续记录已完成事项与待办事项。

### 2.2 模板资产

- [模板/00-模板使用指南.md](./模板/00-模板使用指南.md)
  说明三类官方模板的选择方式与适用边界。
- [模板](./模板/)
  集中存放概念理解类、实战应用类、问题排错类模板。

### 2.3 历史参考资产

- [99-参考与归档](./99-参考与归档/)
  存放已退役规范、旧版迁移方案、历史目录规则与保留参考资料。

说明：

- 主规范文件承担当前有效规则
- 模板目录承担内容生产骨架
- `99-参考与归档` 承担历史保留，不再作为当前规则入口

## 3. 阅读顺序

根据任务类型，建议按以下顺序查阅：

1. 写新文章、改旧文章、补 Frontmatter 时，先看 [01-百科写作规范.md](./01-百科写作规范.md)。
2. 判断文档应放在哪个目录、是否需要 `!MOC`、`README.md` 或 `98-知识库` 时，先看 [02-百科组织规范.md](./02-百科组织规范.md)。
3. 处理旧目录迁移、一级目录拆分与路径重构时，先看 [03-百科迁移方案.md](./03-百科迁移方案.md)。
4. 需要确认当前进度、下一批任务与阶段状态时，只看 [04-重构TODO.md](./04-重构TODO.md)。
5. 开始写正文前，再回到 [模板](./模板/) 选择合适的内容模板。

## 4. 维护约束

维护本目录时，统一遵循以下规则：

- 当前有效规范优先收敛到 `01` 至 `04` 主文件，不重复拆散到多个零碎文档
- `README.md` 只解释本目录如何使用，不替代规范正文
- 历史规则、废弃方案与过渡材料统一迁入 [99-参考与归档](./99-参考与归档/)
- 若新增系统级规则，应优先判断是补充现有主文件，还是确有必要新增独立规范文件

## 5. 使用建议

- 日常维护知识库时，将 `99-系统` 视为“规则源”，而不是“知识源”
- 发生规范冲突时，以主规范文件的最新内容为准
- 执行重构任务时，先核对规则，再修改正文与目录，最后回写 TODO 进度
