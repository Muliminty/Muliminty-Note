# 🐛 问题排错复盘 (Debug)

> **适用场景**：被 Bug 毒打后的含泪复盘。大厂工程师标准的解决 Bug 思路范本。

## 🚨 现象是什么 (Symptom)
> **具体的报错日志（Error log）、复现路径或报错截图。**

(一定要贴出原始的、极具辨识度的报错日志或者 Error Code。这非常重要，因为下次遇到同样的报错，可以通过搜索日志直接命中这篇笔记！)

```log
// 在这里贴报错堆栈或者日志
```

## 🕵️‍♂️ 为什么会这样 (Root Cause Analysis)
> **底层排错链路追踪，以及深层根因探析。**

(在这里复盘你的排坑过程。一开始你以为是什么原因？后来怎么排错发现的？最终触底的根本原因（Root Cause）是什么？比如：原来是因为闭包导致了旧作用域未被释放从而引发了 OOM)

## 🛠️ 怎么解决 (Solution)
> **短期临时方案 (Workaround) + 长期根本修复方案 (Root Cause Fix)。**

(对于阻断性线上 Bug，可能需要分两步走：)

### 方案 A：临时止锚 (Workaround)
(比如：临时重启服务，或者临时加内存顶住。解决燃眉之急的方法)

### 方案 B：彻底修复 (Root Cause Fix)
(比如：修改底层配置文件参数，或者重构泄露点代码，彻底根治 Bug)
