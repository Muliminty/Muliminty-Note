#  语法规范

1. **语法与链接 (原生强兼容)**
   严格使用纯正的 Markdown 语法。文章内部关于其他文章的引用，必须使用标准的原生引用语法（如 `[展示文本](相对路径)`），以保障在不同编辑器（如 Obsidian）之间的兼容互通，并确保后续使用 Quartz 静态部署时链接可用、流转顺畅。

2. **目录与层级 (体系化存储)**
   生成的文档必须极度约束并对齐现有的核心目录树骨架；碎片化、单一释义的概念需统一收敛至专属的 `98-知识库` 中，并打满相关关联标签。

3. **内容创作骨架 (逻辑闭环)**
   拒绝空洞或模糊的陈述。强制基于现有的“官方内容模板”（如：概念理解类、实战应用类、问题排错类）进行高度凝练的垂直输出。

4. **资产稳健命名 (高纯净度保障)**
   新建文档的文件名一律剔除特殊字符与空格，统一使用中划线 `-` 连接词缀或层级关系，严格遵循操作系统与文件系统的通用可扩展排序规则。
   文件名格式
   [序号]-[分类简称]-[中文核心描述].md
   示例：
   01-概念-事件循环机制图解.md
   02-原理-原型与原型链深度剖析.md
   03-实战-手写防抖与节流组件.md
   04-排错-解决跨域Cors拦截问题.md
   05-对比-箭头函数与普通函数差异.md

5. **系统可持续演进 (机器可读)**
   源文格式与排版必须严密且标准化、易于正则提取，为未来可能引入的知识图谱自动化脚本及持续治理打好坚实基础。
