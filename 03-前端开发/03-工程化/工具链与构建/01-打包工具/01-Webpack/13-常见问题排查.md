## 路径解析问题

### 问题：Module not found

**错误信息**：
```
Module not found: Error: Can't resolve './components/Button'
```

**解决方案**：

1. **检查路径是否正确**
```javascript
// ❌ 错误
import Button from './components/Button'

// ✅ 正确
import Button from './components/Button.js'
// 或配置 extensions
```

2. **配置 resolve.extensions**
```javascript
module.exports = {
  resolve: {
    extensions: ['.js', '.jsx', '.json', '.ts', '.tsx']
  }
}
```

3. **配置路径别名**
```javascript
module.exports = {
  resolve: {
    alias: {
      '@': path.resolve(__dirname, 'src'),
      '@components': path.resolve(__dirname, 'src/components')
    }
  }
}
```

## 依赖冲突

### 问题：多个版本的同一依赖

**错误信息**：
```
Module not found: Can't resolve 'react'
```

**解决方案**：

1. **使用 resolve.alias 强制使用指定版本**
```javascript
module.exports = {
  resolve: {
    alias: {
      'react': path.resolve(__dirname, 'node_modules/react')
    }
  }
}
```

2. **检查 package.json**
```bash
npm ls react
```

3. **使用 npm dedupe**
```bash
npm dedupe
```

4. **删除 node_modules 重新安装**
```bash
rm -rf node_modules package-lock.json
npm install
```

## 构建速度慢

### 问题：构建时间过长

**解决方案**：

1. **启用缓存**
```javascript
module.exports = {
  cache: {
    type: 'filesystem'
  }
}
```

2. **减少解析范围**
```javascript
module.exports = {
  resolve: {
    modules: [path.resolve(__dirname, 'src'), 'node_modules']
  },
  module: {
    rules: [
      {
        test: /\.js$/,
        include: path.resolve(__dirname, 'src'),
        exclude: /node_modules/
      }
    ]
  }
}
```

3. **使用多进程**
```javascript
const TerserPlugin = require('terser-webpack-plugin')

module.exports = {
  optimization: {
    minimizer: [
      new TerserPlugin({
        parallel: true
      })
    ]
  }
}
```

4. **使用 DllPlugin**
参考 [性能优化](./09-性能优化.md) 章节。

## 内存溢出

### 问题：JavaScript heap out of memory

**错误信息**：
```
FATAL ERROR: Reached heap limit Allocation failed - JavaScript heap out of memory
```

**解决方案**：

1. **增加 Node.js 内存限制**
```bash
node --max-old-space-size=4096 node_modules/.bin/webpack
```

2. **修改 package.json**
```json
{
  "scripts": {
    "build": "node --max-old-space-size=4096 node_modules/.bin/webpack"
  }
}
```

3. **优化配置**
- 减少入口文件
- 使用代码分割
- 排除不必要的文件

## 样式不生效

### 问题：CSS 样式未加载

**解决方案**：

1. **检查 Loader 配置顺序**
```javascript
// ✅ 正确顺序
use: ['style-loader', 'css-loader', 'sass-loader']
```

2. **检查 sideEffects 配置**
```json
{
  "sideEffects": ["*.css", "*.scss"]
}
```

3. **使用 MiniCssExtractPlugin**
```javascript
const MiniCssExtractPlugin = require('mini-css-extract-plugin')

module.exports = {
  module: {
    rules: [
      {
        test: /\.css$/,
        use: [MiniCssExtractPlugin.loader, 'css-loader']
      }
    ]
  },
  plugins: [
    new MiniCssExtractPlugin()
  ]
}
```

4. **检查 CSS Modules 配置**
```javascript
{
  loader: 'css-loader',
  options: {
    modules: {
      localIdentName: '[name]__[local]__[hash:base64:5]'
    }
  }
}
```

## 热更新不工作

### 问题：HMR 不生效

**解决方案**：

1. **检查 devServer 配置**
```javascript
module.exports = {
  devServer: {
    hot: true
  },
  plugins: [
    new webpack.HotModuleReplacementPlugin()
  ]
}
```

2. **React HMR**
```bash
npm install --save-dev @pmmmwh/react-refresh-webpack-plugin
```

```javascript
const ReactRefreshWebpackPlugin = require('@pmmmwh/react-refresh-webpack-plugin')

module.exports = {
  module: {
    rules: [
      {
        test: /\.(js|jsx)$/,
        use: {
          loader: 'babel-loader',
          options: {
            plugins: ['react-refresh/babel']
          }
        }
      }
    ]
  },
  plugins: [
    new ReactRefreshWebpackPlugin()
  ]
}
```

3. **检查文件监听**
```javascript
module.exports = {
  watchOptions: {
    ignored: /node_modules/
  }
}
```

## 打包文件过大

### 问题：打包后的文件体积过大

**解决方案**：

1. **启用代码分割**
```javascript
module.exports = {
  optimization: {
    splitChunks: {
      chunks: 'all'
    }
  }
}
```

2. **使用 Tree Shaking**
```javascript
module.exports = {
  mode: 'production',
  optimization: {
    usedExports: true,
    sideEffects: false
  }
}
```

3. **压缩代码**
```javascript
const TerserPlugin = require('terser-webpack-plugin')

module.exports = {
  optimization: {
    minimize: true,
    minimizer: [new TerserPlugin()]
  }
}
```

4. **使用 externals**
```javascript
module.exports = {
  externals: {
    'react': 'React',
    'react-dom': 'ReactDOM'
  }
}
```

5. **分析打包结果**
```bash
npm install --save-dev webpack-bundle-analyzer
```

```javascript
const BundleAnalyzerPlugin = require('webpack-bundle-analyzer').BundleAnalyzerPlugin

module.exports = {
  plugins: [
    new BundleAnalyzerPlugin()
  ]
}
```

## 其他常见问题

### Source Map 不工作

**解决方案**：
```javascript
module.exports = {
  devtool: 'source-map'  // 或 'eval-source-map'
}
```

### 图片路径错误

**解决方案**：
```javascript
module.exports = {
  output: {
    publicPath: '/'  // 或 CDN 路径
  }
}
```

### TypeScript 类型错误

**解决方案**：
1. 检查 `tsconfig.json` 配置
2. 确保安装了类型定义
3. 检查 `@types` 包版本

### 端口被占用

**解决方案**：
```javascript
module.exports = {
  devServer: {
    port: 3001  // 更换端口
  }
}
```

## 调试技巧

### 1. 查看详细错误信息

```bash
webpack --stats verbose
```

### 2. 使用 webpack-bundle-analyzer

```bash
npm run build -- --analyze
```

### 3. 检查配置

```javascript
console.log(JSON.stringify(config, null, 2))
```

### 4. 使用 Speed Measure Plugin

```javascript
const SpeedMeasurePlugin = require('speed-measure-webpack-plugin')
const smp = new SpeedMeasurePlugin()

module.exports = smp.wrap({
  // 配置
})
```

## 总结

- **路径问题**：检查路径、配置 alias 和 extensions
- **依赖冲突**：使用 alias 或重新安装
- **构建速度**：启用缓存、多进程、减少解析范围
- **内存溢出**：增加内存限制或优化配置
- **样式问题**：检查 Loader 顺序和 sideEffects
- **热更新**：检查配置和插件
- **文件过大**：代码分割、Tree Shaking、压缩
