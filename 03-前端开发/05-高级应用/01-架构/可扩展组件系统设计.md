# 可扩展组件系统设计

## 1. 关注点

- 组件职责边界
- 设计 token 统一
- 可组合性
- 主题与变体支持

## 2. 设计原则

- 先定义通用能力，再抽象 API
- 避免把业务细节固化进基础组件
