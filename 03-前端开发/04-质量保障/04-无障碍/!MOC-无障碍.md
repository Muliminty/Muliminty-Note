# 无障碍（A11y）MOC

> 前端可访问性设计，确保所有用户都能访问和使用网站。
> 
> **学习路径**：无障碍设计与 [HTML 基础](../../01-Web基础/01-HTML/!MOC-HTML.md) 紧密相关，特别是语义化标签和 ARIA 属性。无障碍测试与 [测试](../01-测试/!MOC-测试.md) 配合使用。

## 📚 核心主题

- [WCAG关键点](./WCAG关键点.md) — WCAG 标准要点（配合 [HTML](../../01-Web基础/01-HTML/!MOC-HTML.md) 使用）
- [键盘导航/屏幕阅读器测试](./键盘导航-屏幕阅读器测试.md) — 可访问性测试（配合 [测试 MOC](../01-测试/!MOC-测试.md) 使用）

## 🎯 核心概念

### WCAG 标准
- 可感知性（Perceivable）
- 可操作性（Operable）
- 可理解性（Understandable）
- 健壮性（Robust）

### 实现要点
- 语义化 HTML（详见 [HTML](../../01-Web基础/01-HTML/!MOC-HTML.md)）
- ARIA 属性（详见 [HTML](../../01-Web基础/01-HTML/!MOC-HTML.md)）
- 键盘导航
- 颜色对比度（配合 [CSS](../../../03-前端开发/01-Web基础/02-CSS/!MOC-CSS.md) 设计）
- 替代文本

### 测试工具
- 屏幕阅读器
- 键盘导航测试（配合 [测试 MOC](../01-测试/!MOC-测试.md)）
- 可访问性检查器
- 颜色对比度检查

---

## 📝 学习建议

1. **前置知识**：需要掌握 [HTML 基础](../../01-Web基础/01-HTML/!MOC-HTML.md)，特别是语义化标签和 ARIA 属性
2. **学习顺序**：HTML → 无障碍基础 → 无障碍实践 → [测试](../01-测试/!MOC-测试.md) 验证
3. **质量保障**：无障碍与 [测试](../01-测试/!MOC-测试.md)、[性能优化](../02-性能/!MOC-性能.md)、[安全](../03-安全/!MOC-前端安全.md) 共同保障代码质量
4. **实践应用**：无障碍设计需要从 [HTML](../../01-Web基础/01-HTML/!MOC-HTML.md) 和 [CSS](../../01-Web基础/02-CSS/!MOC-CSS.md) 设计阶段开始考虑
