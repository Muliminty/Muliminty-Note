# 组件通信

## 1. 常见通信方式

- `Props`：父传子，最基础也最推荐
- `emit`：子传父，适合显式事件回传
- `Provide / Inject`：跨层级透传依赖
- 状态管理：适合跨页面或跨模块共享状态

## 2. 选择原则

- 父子关系明确时，优先使用 `Props + emit`
- 层级过深但作用域仍局部时，可考虑 `Provide / Inject`
- 全局共享状态再引入 `Pinia` 或 `Vuex`

## 3. 常见问题

- 过早把局部状态提升到全局
- 用事件总线替代结构化状态管理
- 组件边界模糊，导致通信路径失控
