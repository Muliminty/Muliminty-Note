## 使用原则

- 优先按主题目录归档，不再直接平铺在根目录。
- 文件命名统一使用规范化技术名与中文描述，避免随意缩写、错误拼写与大小写漂移。
- 当某篇文档已经形成稳定知识域，应迁回对应主目录，而不是长期停留在 `00-经验技巧`。
- 极度零散的临时知识片段，优先收敛到 `98-知识库`。

## 命名约定

- 英文技术名保持标准写法，例如 `Windows`、`Nginx`、`Docker`、`Claude Code`。
- 中英文混排时保留必要空格，例如 `React Context 与自定义 Hooks 架构实战`。
- 避免使用语义含糊的文件名，如“快速上手”“笔记整理”；应尽量体现主题与场景。

## 维护动作

- 新增文档后，同步更新 [!MOC-经验技巧.md](./!MOC-经验技巧.md)。
- 目录调整后，及时修正仓库内相对链接。
- 定期检查是否存在可迁移到主体系或 `98-知识库` 的内容。
