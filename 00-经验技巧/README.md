## 🎯 设计原则

### 1. 按类型分类

内容按照以下类型进行分类：
- **开发技巧**：日常开发中的实用技巧和最佳实践
- **工具与库**：实用的开发工具和第三方库的使用指南
- **问题解决方案**：开发过程中遇到的各种问题的解决方案
- **资源收集**：日常收集的开发资源和学习资料

### 2. 收集箱机制

- **临时收集**：新收集的内容先放入 `收集箱/` 目录
- **定期整理**：整理后将内容移动到相应分类，并更新 MOC 文件
- **保持整洁**：避免收集箱内容过多，定期清理和归类

### 3. 实用性优先

- **聚焦实践**：重点关注实际开发中的问题和解决方案
- **代码示例**：提供完整的代码示例和最佳实践
- **问题驱动**：以解决实际问题为导向

## 📝 添加新内容指南

### 添加新文档

1. **确定分类**：属于哪个分类？（开发技巧、工具库、问题解决方案、资源收集）
2. **选择位置**：
   - 如果是临时收集的内容，先放入 `收集箱/` 目录
   - 如果是已整理的内容，直接放在根目录或相应分类目录
3. **创建文件**：使用规范的文件名（中划线分隔）
4. **更新索引**：在 `!MOC-经验技巧.md` 中添加链接

### 文件命名规范

- **使用中划线分隔**：`前端大文件上传.md`、`iOS-GIF-模糊问题.md`
- **保持简洁清晰**：文件名应准确描述内容
- **使用中文或英文**：保持一致

### 从收集箱整理内容

1. **评估内容**：确定内容属于哪个分类
2. **移动到分类**：将文件移动到相应分类目录（或根目录）
3. **更新链接**：在 MOC 文件中更新链接路径
4. **添加描述**：补充文档描述和标签

## 📋 内容规范

### 内容结构

1. **标题**：清晰明确的标题
2. **概述**：简要描述内容和适用场景
3. **正文**：详细的内容说明
4. **代码示例**：提供完整的代码示例
5. **最佳实践**：总结最佳实践和注意事项
6. **相关链接**：链接到相关文档
7. **标签**：文件末尾添加相关标签

### 内容质量要求

1. **准确性**：确保内容准确无误
2. **完整性**：提供完整的解决方案
3. **实用性**：提供可直接使用的代码示例
4. **清晰性**：表达清晰，易于理解
