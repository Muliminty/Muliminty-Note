# 🛠️ 技能：旧文档优化与重构 (Optimize Old Documents)

## 🎯 技能目标
当你被要求对指定的旧文档文件夹进行优化时，你需要阅读这些文档，并将其重构为符合当前知识库最高标准格式的内容，同时移动/重命名到正确的体系列表中。

## 📋 优化标准与操作要求

本知识库的各种核心规范定义存放在 `99-系统` 目录下。在执行优化之前，你应当优先阅读相关规范文件以将其作为你的工作标准。

### 1. 资产稳健命名与目录规划
请阅读 `99-系统/00-语法规范.md` 和 `99-系统/03-目录规范.md`，确保：
- 资产稳健命名：文件名剔除特殊字符与空格，统一使用中划线 `-` 连接。
- 体系化层级归档：零散碎末知识点收敛汇总，体系化文档正确归档至对应的数字子目录。

### 2. 标准元数据注入 (Frontmatter)
请严格基于 `99-系统/01-元数据规范.md` 的要求，为文档顶部生成或补全完整的 YAML 元数据头（需包含 `moc`, `stage`, `prerequisites` 等体系建设关联项）。

### 3. 内容创作骨架重构 (逻辑闭环)
分析原文档属于哪一种**知识类型**，并强制重构为其官方模板骨架：
> 模板存放于： `99-系统/模板/`

- **概念理解类**（解释名词、架构等）：完全参照 `99-系统/模板/01-概念理解类模板.md`
- **实战应用类**（代码、搭建、配置等）：完全参照 `99-系统/模板/02-实战应用类模板.md`
- **问题排错类**（Bug等）：完全参照 `99-系统/模板/03-问题排错类模板.md`

语言请必须保持“科技博客风格，严谨、去口语化，要点使用列表”。拒绝空洞陈述。

## 🚀 执行流程 (工作流)

当用户说：“优化某某目录”时，你必须遵循以下步骤逐步执行：
1. **先置学习**：如果你还不熟悉当前环境的规范格式，第一时间静默使用 `view_file` 吸取 `99-系统/` 目录下的核心文件要求与模板。
2. **收集信息**：扫描并阅读目标目录下的旧 `.md` 文档。
3. **分析与规划定位**：评估它们属于概念/实战/排错，并拟定它们重命名后的路径位置。
4. **注入与重构**：将重构后的专业内容、YAML头与完美的双链结构写入新路径位置的笔记中。
5. **清理结题**：主动删除或征求意见删除原有旧文件，并告诉用户你的归档逻辑和双链结果。
