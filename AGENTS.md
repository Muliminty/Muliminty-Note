# Codex 项目指南

## 📝 1. 内容与语法规范
- **语言**: 全中文编写，技术术语保留英文。
- **风格**: 科技博客风格，严谨、去口语化，要点使用列表。
- **语法**: 严格遵循纯正 Markdown 语法及双链约束。
  👉 详见 [01-百科写作规范.md](./99-系统/01-百科写作规范.md)
- **元数据**: 每一篇文章都必须包含标准的 YAML Frontmatter，以保证机器检索的高效性和知识网络的稳固。
  👉 详见 [01-百科写作规范.md](./99-系统/01-百科写作规范.md)

---

## 📄 2. 核心笔记模板
生成具体的知识内容时，必须根据知识类型选择相应的**官方垂类模板**进行垂直化输出：

- 🎯 **概念理解类**：用于解释定义、原理、架构图等理论知识。
  👉 参考：[01-概念理解类模板.md](./99-系统/模板/01-概念理解类模板.md)
- 🛠️ **实战应用类**：用于记录代码实现、环境配置、最佳实践等干货。
  👉 参考：[02-实战应用类模板.md](./99-系统/模板/02-实战应用类模板.md)
- 🐛 **问题排错类**：用于记录 Bug 复现、日志分析、根因探析与修复方案。
  👉 参考：[03-问题排错类模板.md](./99-系统/模板/03-问题排错类模板.md)

## 🚫 5. 受保护文件清单
以下文件由 Quartz 静态站点框架依赖，**严禁任何 AI Agent 修改、删除或重命名**：

- `quartz.config.ts` — Quartz 站点核心配置
- `index.md` — Quartz 站点首页入口

任何涉及上述文件的变更需求，必须由人工手动操作，AI 仅可提出建议但不得执行写入。
