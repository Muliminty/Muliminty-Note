# 知识库 MOC

## 定位

`98-知识库/` 是整个知识库的**永久概念卡片库**。

核心职责：

- 收纳独立概念卡片，作为知识网络的永久节点
- 提供低摩擦的快速写入通道
- 通过双向引用与正式专题文章互联，融入整体知识体系

不适合放在这里的内容：

- 已明确属于某个正式专题的系统性知识 → 直接写入对应专题目录
- 资源索引、书签收藏 → 放入 `09-工具与效率/03-资源收藏/`
- 历史草稿、旧版产物 → 放入对应专题的 `99-参考与归档/`

## 写入规范

### Frontmatter（精简模式）

仅需 3 个必填字段，详见 [01-百科写作规范.md §3.3](../99-系统/01-百科写作规范.md)：

```yaml
---
title: "核心概念名"
date: "YYYY-MM-DD"
tags: ["领域标签"]
---
```

### 文件命名

```text
YYYYMMDD-领域-核心概念.md
```

示例：

- `20260315-网络-HTTP缓存协商机制.md`
- `20260407-架构-幂等设计模式.md`
- `20260509-效率-代码审查检查清单.md`

### 正文模板

根据知识类型选择对应模板：

- 概念理解 → [01-概念理解类模板.md](../99-系统/模板/01-概念理解类模板.md)
- 实战应用 → [02-实战应用类模板.md](../99-系统/模板/02-实战应用类模板.md)
- 问题排错 → [03-问题排错类模板.md](../99-系统/模板/03-问题排错类模板.md)

## 双向引用

每篇概念卡片末尾应添加 `## 关联知识` 区块，链接到相关的正式专题文章。Obsidian 反向链接面板会自动在正式文章侧显示引用关系，无需手动修改正式文章。

## 迁入正式专题（可选）

当同一领域的原子笔记积累到足以形成专题时，**可选择**迁入正式专题目录：

1. 在对应一级目录下创建或选择专题目录
2. 将相关原子笔记迁入，补齐完整 Frontmatter
3. 按 `[两位序号]-[主题名].md` 重命名
4. 补齐 `!MOC` 和 `README.md`

概念卡片也可以不迁移，永久保留在 `98-知识库/`，通过关联知识链接融入体系即可。
