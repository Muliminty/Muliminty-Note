# 知识库 MOC

## 定位

`98-知识库/` 是整个知识库的**低门槛原子知识缓冲区**。

核心职责：

- 收纳尚未确定长期归属的独立知识点
- 提供"先落盘、后归类"的快速写入通道
- 降低日常学习和对话沉淀的写入摩擦

不适合放在这里的内容：

- 已明确属于某个正式专题的系统性知识 → 直接写入对应专题目录
- 资源索引、书签收藏 → 放入 `09-工具与效率/03-资源收藏/`
- 历史草稿、旧版产物 → 放入对应专题的 `99-参考与归档/`

## 写入规范

### Frontmatter（精简模式）

仅需 3 个必填字段，详见 [01-百科写作规范.md §3.3](../99-系统/01-百科写作规范.md)：

```yaml
---
title: "核心概念名"
date: "YYYY-MM-DD"
tags: ["领域标签"]
---
```

### 文件命名

```text
YYYYMMDD-领域-核心概念.md
```

示例：

- `20260315-网络-HTTP缓存协商机制.md`
- `20260407-架构-幂等设计模式.md`
- `20260509-效率-代码审查检查清单.md`

### 正文模板

根据知识类型选择对应模板：

- 概念理解 → [01-概念理解类模板.md](../99-系统/模板/01-概念理解类模板.md)
- 实战应用 → [02-实战应用类模板.md](../99-系统/模板/02-实战应用类模板.md)
- 问题排错 → [03-问题排错类模板.md](../99-系统/模板/03-问题排错类模板.md)

## 升级路径

当同一领域的原子笔记积累到 3-5 篇时，应考虑将其迁入正式专题目录：

1. 在对应一级目录下创建或选择专题目录
2. 将相关原子笔记迁入，补齐完整 Frontmatter
3. 按 `[两位序号]-[主题名].md` 重命名
4. 补齐 `!MOC` 和 `README.md`
