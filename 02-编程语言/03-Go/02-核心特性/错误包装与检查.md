# 错误包装与检查

## 1. 核心能力

Go 通过 `%w`、`errors.Is`、`errors.As` 支持错误链路的包装与识别。

## 2. 常见用法

```go
if err != nil {
    return fmt.Errorf("读取配置失败: %w", err)
}
```

## 3. 判断方式

- `errors.Is`：判断是否包含某个特定错误
- `errors.As`：提取某种具体错误类型
- `errors.Unwrap`：获取下一层被包装错误
