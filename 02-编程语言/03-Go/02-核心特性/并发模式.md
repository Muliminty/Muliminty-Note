# 并发模式

## 1. 常见模式

- Producer-Consumer
- Worker Pool
- Fan-out / Fan-in
- Pipeline
- Context 驱动的取消协作

## 2. 设计目标

- 降低 Goroutine 之间的耦合度
- 控制并发数量
- 保证任务退出路径可控

## 3. 使用建议

- 优先明确输入、输出与退出信号
- 优先考虑是否需要限流或超时
- 避免 Goroutine 泄漏
